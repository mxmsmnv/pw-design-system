## Workflow

- Edit source files under `src/`, `assets/css/`, and `assets/js/`.
- Do not hand-edit `index.html` or `Draft-AdminThemeUikit-DesignSystem-UIKit.html`; they are generated by `scripts/build.js`.
- Run `npm run build` after source edits.
- Keep component fixes targeted. If the issue is with buttons, edit `src/components/04-buttons.html` and only touch shared CSS when the fix applies globally.
- Preserve native ProcessWire/AdminThemeUikit naming and markup where known.
- Prefer real ProcessWire Inputfield markup and UIkit classes over invented helper classes.
- Use `--pw-*` variables for colors, spacing, and theme-aware styling.
- Keep this kit dependency-free unless there is a clear reason to add tooling.

## Component Map

- `01-accents.html` - accent color controls.
- `04-buttons.html` - UIkit buttons and ProcessWire save dropdown pattern.
- `05-forms.html` - UIkit form elements and `uk-fieldset`.
- `10-cards.html` - card variants.
- `24-inputfields.html` - ProcessWire `InputfieldWrapper` examples.
- `25-module-guidelines.html` - guidance for module authors.
- `34-comment.html` - UIkit comment example.
- `36-overrides.html` - Konkat theme files, settings, and CSS snippets.

## Verification

At minimum, run:

```sh
npm run build
git diff --check
```

If visual behavior changes, open `index.html` in a browser and check both light and dark themes.
